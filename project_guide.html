CodeZen Backend Guide (Beginner Friendly)

This document explains how every part is connected: FastAPI endpoints, Celery queue, Redis, Docker sandbox, file dependencies, and key functions.

1) Project Snapshot

FastAPI Celery Redis Docker Sandbox Multi-language Execution

User submits source code and input to the API. API puts the job into Celery queue. Worker takes job, runs it inside isolated Docker container, and stores result. Client gets output using task ID.

2) Core Terminologies

Task: A background job. Here, run_code_task is the task.
Queue: Waiting line of jobs that workers process.
Broker: Message hub that carries tasks from API to workers. Here, Redis.
Result Backend: Place where task result is stored. Here, Redis.
Worker: Celery process that executes queued tasks.
Sandbox: Controlled container where user code runs with limits.
Timeout: Max run time. If exceeded, code is killed.
Stdout/Stderr: Normal output / error output from program.
Exit Code: Process result code. 0 means success.

5) Request Flows

5.1 Submit + Poll Flow

sequenceDiagram
    participant Client
    participant API as FastAPI main.py
    participant Redis as Redis Broker/Backend
    participant Worker as Celery Worker
    participant Exec as executor.py
    participant Docker as Docker Sandbox

    Client->>API: POST /submit {language, code, stdin}
    API->>Redis: run_code_task.delay(payload)
    API-->>Client: {task_id, status: queued}

    Worker->>Redis: consume queued task
    Worker->>Exec: execute_code(...)
    Exec->>Docker: create/run sandbox container
    Docker-->>Exec: output logs + status
    Exec-->>Worker: result dict
    Worker->>Redis: store result

    Client->>API: GET /result/{task_id}
    API->>Redis: fetch AsyncResult(task_id)
    API-->>Client: queued/running/completed/failed

5.2 Execute-And-Wait Flow

flowchart TD
    A[POST /execute] --> B[Validate body via ExecuteRequest]
    B --> C[Queue run_code_task]
    C --> D{Result within wait_timeout?}
    D -- Yes --> E[Return completed + result]
    D -- No --> F[Return queued + task_id]
    F --> G[Client polls /result/task_id]

6) Executor Internal Flow

flowchart TD
      A[Start execute code] --> B[Normalize language]
      B --> C{Language supported}
      C -- No --> Z[Return unsupported language error]
      C -- Yes --> D[Load language config]
      D --> E[Create Docker client]
      E --> F[Create locked down container]
      F --> G[Build archive with source and input]
      G --> H[Copy archive into sandbox]
      H --> I[Start container]
      I --> J{Finished before timeout}
      J -- Yes --> K[Read exit status]
      J -- No --> L[Kill container and mark timeout]
      K --> M[Collect logs]
      L --> M
      M --> N[Truncate large output]
      N --> O[Remove container and close client]
      O --> P[Return result]

7) Key Functions and Their Basic Job

File Function / Object What It Does (Simple)
main.py root() Returns base service message.
main.py health() Simple health check endpoint.
main.py submit_execution(request) Queues job and returns task ID immediately.
main.py get_result(task_id) Reads Celery task state/result and returns status.
main.py execute_and_wait(request) Queues job and waits for quick completion.
schemas.py ExecuteRequest Validates input fields and limits.
schemas.py validate_language(value) Normalizes language and blocks unsupported values.
schemas.py ExecutionResult / TaskResultResponse Defines output format returned to clients.
celery_app.py celery_app = Celery(...) Creates Celery app using Redis broker/backend.
tasks.py run_code_task(payload) Celery task wrapper that calls executor.
executor.py normalize_language(language) Maps alias like js to javascript.
executor.py _build_archive(source, code, stdin) Packs source file and stdin into tar bytes.
executor.py _collect_logs(container) Reads stdout and stderr from container.
executor.py _truncate_output(text) Limits output size to avoid huge responses.
executor.py execute_code(...) Full sandbox run pipeline and final result creation.

8) Environment and Runtime Dependencies

Python Dependencies
fastapi, uvicorn, celery[redis], docker SDK
Infra Services
Redis container, API container, Worker container
Sandbox Image Tools
Python3, Node.js, Java JDK, GCC/G++
Host Requirement
Docker Engine must be running

Note: API and Worker containers mount the Docker socket so Worker can create sandbox containers dynamically.
